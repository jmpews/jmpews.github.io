    前言

    这篇文章主要会介绍到, frida-gum 在 trampoline, 调度器, 入口点跳转, 指令修复做的一些细节工作.

    Hook点构建流程

    首先明确几个处理点:

    1. 构建进入*函数执行前处理的跳板 (enter_trampoline)
    2. 构建调用原函数的跳板 (invoke_tarmpoline), 涉及到指令修复.
    3. 构建进入 函数执行后处理的跳板 (leave_trampoline)
    4. 构建入口点的跳转, 激活该 Hook 点.

    下面就针对每个点说一下.

    函数执行前处理 跳板

    这个跳板, 是从入口点跳过来的. 这里跳的方式值得深究.

    绝对地址跳. 缺点: 需要多条指令,无法 hook 单指令函数. 优点: 可以任意跳

    相对地址跳. 缺点: 需要在入口点周围找到 b xxx 范围内的可用的内存块存放跳转指令. 优点: 单指令的 hook.

    指令修复 - relocator

    因为需要对于 hook 的入口点写入跳转指令, 所以需要把之前的指令拷贝我们自己分配的内存块. 这里存在一个问题, 假如存在 b xxx 类似指令, 需要依赖于 pc 寄存器的, 由于被修复的指令的地址发生改变. 所以需要对原指令进行修复.

    修复思路: 相对地址改为绝对地址.
