在做安全开发遇到各种调试, 这涉及到很多调试的技巧

调试的奇淫技巧

主动触发调试中断

首先推荐一个库: https://github.com/scottt/debugbreak

在 C/C++ 调试过程中不像 pyhton 那么简单, 直接 import pdb;pdb.set_trace() 就可以, 所以希望找到主动触发中断的技巧, 其实也就是触发调试陷阱(Trap), 这里大概有三种方法.

其中 1 和 3 对应汇编一条指令

使用 __builtin_trap(), 触发中断后直接使用 0x90 填充, 在 lldb 中是 memory write -s 2 address 0x9090

使用 raise(SIGTRAP);, 使用之前需要 #include <signal.h>, 触发中断后需要使用 finish 返回上层栈.

使用 asm("int3");, 仅仅会触发中断, 之后便正常执行.
