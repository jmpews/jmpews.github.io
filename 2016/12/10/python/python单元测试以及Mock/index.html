    前言

    • 测试可以保证你的代码在一系列给定条件下正常工作(e:保证基本功能正确)
    • 测试允许人们确保对代码的改动不会破坏现有的功能(e:保证在修改的过程中,功能不变)
    • 测试迫使人们在不寻常条件的情况下思考代码，这可能会揭示出逻辑错误(e:传入不同参数或边界参数)
    • 良好的测试要求模块化，解耦代码，这是一个良好的系统设计的标志(e:模块化设计)
